About me

Hi. I'm John Lynch. I love coding, playing with mathematics, and using both of these to create digital art. I also love playing music (piano, sax, tin whistles), growing vegetables, trees and even a few flowers; cooking and making chutneys, marmalades etc.; and enjoying good coffee, preferably freshly-ground and ideally not long roasted!

Back in the day, I did a mathematics degree at Hull University, since which I've worked with computers for many years in a variety of roles, mostly involving operations and technical support, as well as running a PC factory, but over the past few years my early passion for coding has, delightfully, returned. And I've been using it, and my mathematical knowledge, to create digital art, mainly using Java and Clojure, but lately also in GLSL, directly programming the GPU.

All this stuff is quite specialised, though, so in January 2018 I enrolled on a Code Institute course to learn all the technologies required to develop both the front- and back-ends of websites and web applications, and to understand all the necessary wiring that serves to knit all these technologies together.

These days I'm living in the wild hills of West Cork, Ireland, but over the years have worked in various parts of the UK, Ireland, Australia, Spain, Italy and Thailand.

About this website

This website was lovingly crafted by hand, using raw semantic HTML5 and CSS3. It has formed the substance of the first "milestone project" I have had to complete as part of my studies on the Full-stack Developer course at Code Institute in Dublin.

Technical Info

I did not use any preprocessors like Sass or HAML, or CSS libraries like Bootstrap, as I felt I first needed to learn the nuts and bolts, particularly of CSS. The only external resources used are Google Fonts (for Mali and Rubik), Font Awesome for social media icons, and embedded iframes from Youtube and Shadertoy, which is all my own content in the form of fractal videos and shader programs I've created myself.

All the images are mine. The photographs were taken with my Nikon Coolpix P500 digital SLR camera, and the fractal images were generated by a Java escape-time fractal program which I have been developing over the past couple of years. I use ImageMagick to resize them where necessary, or occasionally GIMP.

I produce the videos from an escape-time-fractal-image-producing program I have been developing over the past couple of years. By iterating the Julia sets of compositions of complex trig. functions and varying the constant, I produce image sequences. I add title sequences with another Java program of mine, and then pass them to ImageJ to turn into an .avi video, then through VLC Player to compress to a more manageable .mp4 file, ready for uploading to Youtube.

I write the articles in Markdown and use a web service like https://markdowntohtml.com/ to convert to HTML, then edit the HTML if necessary. For version control I use Git on my local machine. For vendor-specific prefixes I used Autoprefixer CSS online.

I used CSS Grid to lay out the content. I experimented by building dummy pages while watching Rachel Andrew's and Wes Bos's tutorials. Together with using viewport units, CSS Grid seems to make it easier to make the site responsive and to make elements behave themselves when the user resizes the viewport.

I used Sublime Text 3 in dual-pane mode to code it on my local machine, with the editor on the laptop screen and the site displaying in a browser window on an external monitor, using Ctrl+s Alt+Tab F5 to see a refreshed page in the browser. It was tested continually as I developed it, in Firefox Developer Edition v.63.0 (64-bit) and Chromium v.69.0, using both Firefox and Chromium developer tools, and nearing the conclusion of the project also on other physical devices.

Ongoing testing includes Safari, Opera, Edge, IE and eventually, text-based browsers. I also intend to research meta-tags etc. and SEO in general in order to better attract visitors.

Purpose & Functionality

  • to present some of my digital art creations to the world;
  • to fulfil the requirements of the first "milestone project" of my Code Institute course;
  • to be an online shed where I can store things, try out new tools in real-online-world ways;
  • eventually to be a sales portal for physical renderings of my digital art.

It is, I hope, structured in an intuitive way, with a main navigation bar at the top, footer at the bottom and content in two areas in between. The styling is consistent through all pages. Links are styled to change colour when hovered upon, and external links open in a new tab. The main navigation links also scale up very slightly in size to encourage the user to click them!

The footer contains icons for Twitter, Github, LinkedIn and Shadertoy, which link as expected to my accounts on the respective platforms. The two content panels are displayed next to each other between header and footer on desktop, while on mobile in portrait mode all areas are in one column.
